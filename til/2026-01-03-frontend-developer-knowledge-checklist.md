# 프론트엔드 개발자 지식 체크리스트

본 문서는 프론트엔드 개발자가 실무 및 성장 과정에서 점검해야 할 핵심 지식을 **상황(시나리오) 기반**으로 정리한 체크리스트입니다.

## 사용 방법

- **옵션 A (3단계 체크)**: 각 항목을 `Lv1(설명 가능) / Lv2(실습 경험) / Lv3(실무 해결 경험)`으로 점검합니다.
- **옵션 B (트랙 체크)**: 목표에 따라 `취업 면접용 / 실무 적응용 / 시니어 성장용`에서 필요한 범위를 우선 점검합니다.

## 옵션 A) 3단계 체크 기준

- **Lv1 (설명 가능)**: 개념, 원리, 트레이드오프를 말/글로 명확히 설명할 수 있습니다.
- **Lv2 (실습 경험)**: 작은 예제/토이/과제로 직접 구현해본 경험이 있습니다.
- **Lv3 (실무 해결 경험)**: 실제 서비스에서 문제를 진단 → 원인 규명 → 수정 → 검증까지 수행해본 경험이 있습니다.

---

## A-1. 화면/렌더링 (렌더링/레이아웃/스크롤)

- **스크롤 성능 이슈(버벅임)**
  - [ ] **Lv1**: DOM/CSSOM, Layout/Paint/Composite 차이를 설명할 수 있습니다.
  - [ ] **Lv2**: 성능에 유리한 애니메이션 속성(transform/opacity) 중심으로 구성해본 적이 있습니다.
  - [ ] **Lv3**: Performance 프로파일링을 통해 병목을 찾아 수정하고 개선을 검증해본 적이 있습니다.
- **레이아웃 점프(CLS)**
  - [ ] **Lv1**: CLS가 발생하는 대표 원인을 설명할 수 있습니다.
  - [ ] **Lv2**: 이미지/폰트/스켈레톤/placeholder로 레이아웃을 안정화해본 적이 있습니다.
  - [ ] **Lv3**: 실제 페이지의 CLS를 개선하고 지표 변화로 확인해본 적이 있습니다.
- **모달/오버레이 UX**
  - [ ] **Lv1**: 모달에서 focus/스크롤/배경 인터랙션 관리가 왜 필요한지 설명할 수 있습니다.
  - [ ] **Lv2**: 스크롤 락, focus trap/restore를 구현해본 적이 있습니다.
  - [ ] **Lv3**: 접근성/모바일 환경 이슈까지 포함해 모달 관련 버그를 해결해본 적이 있습니다.
- **z-index/겹침 문제**
  - [ ] **Lv1**: stacking context를 설명할 수 있습니다.
  - [ ] **Lv2**: 레이어 구조를 재설계하여 겹침 문제를 해결해본 적이 있습니다.
  - [ ] **Lv3**: 복잡한 레이아웃/포털/고정 헤더 환경에서 일관되게 해결해본 적이 있습니다.

## A-2. 상태/React (렌더링, 훅, 상태 설계)

- **useEffect 의존성/무한 루프**
  - [ ] **Lv1**: 의존성 배열과 stale closure를 설명할 수 있습니다.
  - [ ] **Lv2**: 의존성을 올바르게 구성하고 불필요한 effect를 줄여본 적이 있습니다.
  - [ ] **Lv3**: 실서비스에서 무한 루프/경합(race) 문제를 추적하여 해결해본 적이 있습니다.
- **리렌더 과다**
  - [ ] **Lv1**: 리렌더가 발생하는 조건을 설명할 수 있습니다.
  - [ ] **Lv2**: 메모이제이션(useMemo/useCallback/memo)을 목적에 맞게 적용해본 적이 있습니다.
  - [ ] **Lv3**: React Profiler로 병목을 확인하고 개선을 검증해본 적이 있습니다.
- **상태 경계(로컬/전역/서버 상태)**
  - [ ] **Lv1**: 로컬/전역/서버 상태를 구분하는 기준을 설명할 수 있습니다.
  - [ ] **Lv2**: 서버 상태 캐싱/동기화(예: TanStack Query)를 적용해본 적이 있습니다.
  - [ ] **Lv3**: 전역 상태 과다/props drilling을 구조적으로 개선해본 적이 있습니다.
- **리스트 key/입력값 튐**
  - [ ] **Lv1**: key 안정성이 중요한 이유를 설명할 수 있습니다.
  - [ ] **Lv2**: 안정적인 key 전략을 적용해본 적이 있습니다.
  - [ ] **Lv3**: 실제 버그를 재현/수정/회귀 방지까지 해본 적이 있습니다.

## A-3. API/네트워크 (데이터 페칭, 캐시, 오류)

- **로딩/에러/빈 상태 처리**
  - [ ] **Lv1**: 사용자 경험 관점에서 세 상태의 필요성을 설명할 수 있습니다.
  - [ ] **Lv2**: 화면에서 일관된 상태 처리 패턴을 구현해본 적이 있습니다.
  - [ ] **Lv3**: 장애 상황에서 사용자 영향 최소화 UX까지 설계/개선해본 적이 있습니다.
- **요청 취소/중복 요청 방지**
  - [ ] **Lv1**: AbortController 및 경합 문제를 설명할 수 있습니다.
  - [ ] **Lv2**: 라우트 이동/입력 변경 중 요청 취소를 적용해본 적이 있습니다.
  - [ ] **Lv3**: 중복 요청/경합으로 인한 데이터 오염을 실무에서 해결해본 적이 있습니다.
- **CORS / preflight**
  - [ ] **Lv1**: CORS와 preflight가 발생하는 조건을 설명할 수 있습니다.
  - [ ] **Lv2**: 개발 환경에서 CORS 문제를 원인 파악하고 해결해본 적이 있습니다.
  - [ ] **Lv3**: 배포 환경에서 프록시/헤더/캐시까지 고려하여 안정적으로 해결해본 적이 있습니다.
- **캐시/신선도**
  - [ ] **Lv1**: Cache-Control, ETag 개념을 설명할 수 있습니다.
  - [ ] **Lv2**: 캐시로 인한 최신 데이터 문제를 회피/개선해본 적이 있습니다.
  - [ ] **Lv3**: 성능과 신선도 균형을 지표 기반으로 운영해본 적이 있습니다.

## A-4. 인증/보안 (실무 필수 안전장치)

- **인증 방식 이해**
  - [ ] **Lv1**: 쿠키/토큰 기반 인증의 차이를 설명할 수 있습니다.
  - [ ] **Lv2**: 401 처리, 로그인 유지(세션/토큰), refresh 흐름을 구현해본 적이 있습니다.
  - [ ] **Lv3**: 만료 경합/루프/보안 사고성 이슈를 실무에서 해결해본 적이 있습니다.
- **XSS/CSRF**
  - [ ] **Lv1**: XSS/CSRF의 원리와 방어 관점을 설명할 수 있습니다.
  - [ ] **Lv2**: 위험한 렌더링(HTML 삽입) 회피 및 입력/출력 방어를 적용해본 적이 있습니다.
  - [ ] **Lv3**: 실제 서비스에서 보안 이슈를 점검/개선/가이드화해본 적이 있습니다.

## A-5. 폼/입력 (검증, 업로드, 엣지 케이스)

- **controlled/uncontrolled 및 검증**
  - [ ] **Lv1**: 두 방식의 차이와 선택 기준을 설명할 수 있습니다.
  - [ ] **Lv2**: 클라이언트/서버 validation을 함께 구성해본 적이 있습니다.
  - [ ] **Lv3**: 복잡한 폼(다단계/동적 필드)에서 안정적으로 운영해본 적이 있습니다.
- **파일 업로드**
  - [ ] **Lv1**: 파일 업로드에서 UX(프로그레스/취소/검증)가 중요한 이유를 설명할 수 있습니다.
  - [ ] **Lv2**: progress 표시, 취소, 실패 재시도 UX를 구현해본 적이 있습니다.
  - [ ] **Lv3**: 대용량/모바일/네트워크 불안정 환경 이슈를 해결해본 적이 있습니다.
- **IME/모바일 입력**
  - [ ] **Lv1**: IME 입력에서 발생할 수 있는 이벤트/상태 이슈를 설명할 수 있습니다.
  - [ ] **Lv2**: 관련 버그를 재현하고 우회/개선해본 적이 있습니다.
  - [ ] **Lv3**: 다양한 디바이스/브라우저에서 회귀 테스트까지 해본 적이 있습니다.

## A-6. 접근성(A11y)

- **시맨틱/키보드 내비게이션**
  - [ ] **Lv1**: 시맨틱 태그와 키보드 접근성의 중요성을 설명할 수 있습니다.
  - [ ] **Lv2**: focus 관리(모달, 드롭다운)와 aria 적용을 해본 적이 있습니다.
  - [ ] **Lv3**: 스크린리더 기준으로 결함을 찾아 개선해본 적이 있습니다.

## A-7. 스타일/디자인 시스템

- **토큰/테마/variant**
  - [ ] **Lv1**: 디자인 토큰(CSS 변수) 기반 설계의 장점을 설명할 수 있습니다.
  - [ ] **Lv2**: variant(크기/상태/색상) API를 설계해본 적이 있습니다.
  - [ ] **Lv3**: 확장성/일관성 문제를 구조적으로 해결해본 적이 있습니다.
- **CSS 스코프/우선순위**
  - [ ] **Lv1**: 우선순위/특이성/전역 오염을 설명할 수 있습니다.
  - [ ] **Lv2**: CSS Modules 등으로 스코프 문제를 관리해본 적이 있습니다.
  - [ ] **Lv3**: 대규모에서 스타일 충돌을 줄이는 규칙/가이드를 운영해본 적이 있습니다.

## A-8. 성능 (측정 → 개선 → 검증)

- **Core Web Vitals**
  - [ ] **Lv1**: LCP/INP/CLS 의미를 설명할 수 있습니다.
  - [ ] **Lv2**: 이미지/폰트/코드 스플리팅으로 개선을 적용해본 적이 있습니다.
  - [ ] **Lv3**: 지표 기반으로 개선을 설계하고 회귀 방지까지 해본 적이 있습니다.

## A-9. 에러 처리/관측성

- **에러 UX/에러 바운더리/트래킹**
  - [ ] **Lv1**: 사용자 메시지/로깅/복구 UX를 구분해 설명할 수 있습니다.
  - [ ] **Lv2**: 에러 바운더리 및 에러 트래킹 도구를 연동해본 적이 있습니다.
  - [ ] **Lv3**: 재현 어려운 이슈를 로그/트레이싱으로 추적해 해결해본 적이 있습니다.

## A-10. 테스트

- **테스트 전략**
  - [ ] **Lv1**: 단위/컴포넌트/E2E 테스트 목적과 비용을 설명할 수 있습니다.
  - [ ] **Lv2**: 핵심 플로우에 대해 테스트를 작성해본 적이 있습니다.
  - [ ] **Lv3**: flaky 테스트 제거/CI 안정화/리팩토링 안전망 운영 경험이 있습니다.

## A-11. 빌드/배포/환경

- **번들/캐시/소스맵**
  - [ ] **Lv1**: 번들링, 트리 쉐이킹, 소스맵의 의미를 설명할 수 있습니다.
  - [ ] **Lv2**: 환경변수(dev/stage/prod) 분리 및 배포 흐름을 다뤄본 적이 있습니다.
  - [ ] **Lv3**: 배포 후에만 발생하는 문제(CDN/캐시 등)를 디버깅해 해결해본 적이 있습니다.

## A-12. 협업

- **PR/리뷰/문서화**
  - [ ] **Lv1**: 좋은 PR 단위와 리뷰 관점을 설명할 수 있습니다.
  - [ ] **Lv2**: README/가이드 문서를 작성해본 적이 있습니다.
  - [ ] **Lv3**: 팀의 생산성을 높이는 컨벤션/자동화를 주도해본 적이 있습니다.

## 옵션 B) 트랙별 체크리스트

### B-1. 취업 면접용 (설명력 중심)

- [ ] **웹 동작 원리**: 렌더링 파이프라인, 이벤트 루프, CORS
- [ ] **JavaScript 핵심**: 클로저, this, 비동기, 에러 처리
- [ ] **React 핵심**: 상태 설계, useEffect 의존성, 리스트 key, 성능 기본
- [ ] **CSS 핵심**: flex/grid, stacking context, 반응형
- [ ] **보안/성능**: XSS/CSRF, Core Web Vitals 개념
- [ ] **테스트 개념**: 단위/통합/컴포넌트/E2E 목적 구분

### B-2. 실무 적응용 (바로 일하기)

- [ ] **데이터 페칭 운영**: 로딩·에러·빈 상태, 취소, 중복요청 방지, 캐시
- [ ] **폼 실무**: validation, 서버 에러 매핑, 파일 업로드
- [ ] **라우팅/권한**: 보호 라우트, 401 처리, 리다이렉트
- [ ] **UI 품질**: 모달 스크롤/포커스, 접근성 기본, 반응형
- [ ] **디버깅 루프**: 재현 → 로그 → 가설 → 검증
- [ ] **배포 이슈 대응**: 환경변수, 캐시, 소스맵 기반 추적

### B-3. 시니어 성장용 (설계/운영/확장)

- [ ] **아키텍처/모듈 경계**: 의존성 관리, 도메인/뷰/인프라 분리
- [ ] **디자인 시스템**: 토큰/variant/컴포넌트 API, 접근성 내장
- [ ] **성능 엔지니어링**: 지표 기반 개선, 관측성, 회귀 방지
- [ ] **보안/리스크 관리**: 위협 모델, 공급망, PII/로그 정책
- [ ] **테스트/CI 전략**: 테스트 피라미드, flaky 관리, 파이프라인 최적화
- [ ] **팀 생산성**: 리뷰 문화, 문서, 자동화, 기술부채 관리
